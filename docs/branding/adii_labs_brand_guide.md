## 1. Brand Positioning

**ADii Labs** is a software product studio that builds **professional-grade, offline-first business tools** for freelancers, small businesses, and micro-enterprises in emerging markets.

### Positioning Statement

> _"ADii Labs builds beautiful, reliable business software that works everywhere — even without the internet."_

### Market Position
- **Not** an enterprise software company
- **Not** a generic app factory
- **Is** a focused product studio with a cohesive ecosystem
- **Is** a champion of offline-first, mobile-first business tools

## 4. Product Principles

Every ADii Labs product must adhere to these non-negotiable principles:

### 1. Offline-First
> The app must work flawlessly without internet. Cloud features are enhancements, not requirements.

### 2. Beautiful by Default
> No ugly apps. Every screen, every interaction, every output must feel premium.

### 3. Zero Configuration
> Users should be productive within 60 seconds. No complex setup, no learning curve.

### 4. Data Ownership
> Users own their data. Export everything. Lock-in nothing. Transparent storage.

### 5. Professional Output
> PDFs, reports, and exports must be indistinguishable from desktop-generated documents.

### 6. Multilingual Native
> Not an afterthought. RTL, locale-aware formatting, and cultural sensitivity from day one.

## 6. Tone of Voice

### Personality Traits

| Trait | Description | Example |
|---|---|---|
| **Professional** | Competent, trustworthy, expert | "Your invoice has been saved." |
| **Friendly** | Warm, approachable, helpful | "Welcome back! Ready to create an invoice?" |
| **Clear** | Simple, direct, no jargon | "Add your business logo" (not "Upload brand asset") |
| **Encouraging** | Positive, supportive | "Great! Your first invoice is ready." |

### Writing Guidelines

| Do | Don't |
|---|---|
| Use simple, direct language | Use technical jargon |
| Be concise (< 20 words per message) | Write long paragraphs |
| Use active voice | Use passive voice |
| Be specific about actions | Be vague or ambiguous |
| Show empathy in error messages | Blame the user |

### Error Message Format
```
What happened: [Clear description]
What to do: [Actionable next step]
```

**Example:**
> ❌ "Error: Database operation failed (code: 0x3F)"  
> ✅ "Couldn't save the invoice. Please try again, or restart the app if the issue continues."

## 7. App Naming System

ADii Labs products follow a consistent naming convention:

```
[Function]Flow [Edition]
```

### Naming Rules
1. **Function** describes the core purpose (Invoice, Ledger, POS, etc.)
2. **Flow** is the ADii Labs product suffix — represents smooth, effortless workflow
3. **Edition** (optional) indicates the tier (Pro, Lite, Business)

### Examples

| Name | Purpose |
|---|---|
| **InvoiceFlow Pro** | Professional invoicing app |
| **InvoiceFlow Lite** | Free/limited invoicing app (if created) |
| **LedgerFlow** | Bookkeeping and accounting |
| **POSFlow** | Point of sale system |
| **InventoryFlow** | Stock and inventory management |
| **ClientFlow** | CRM and client management |
