## Audit Guidelines

### Severity Levels

| Level | Label | Description |
|---|---|---|
| **P0** | 🔴 Critical | App crashes, data loss, or complete feature failure |
| **P1** | 🟠 High | Major functionality broken, no workaround |
| **P2** | 🟡 Medium | Feature partially broken, workaround exists |
| **P3** | 🔵 Low | Minor visual issue, cosmetic defect |
| **P4** | ⚪ Trivial | Suggestion, enhancement, or nitpick |

### Issue Types

| Code | Type |
|---|---|
| **BUG** | Functional bug |
| **UI** | Visual/layout issue |
| **UX** | Usability problem |
| **PERF** | Performance issue |
| **L10N** | Localization issue |
| **A11Y** | Accessibility issue |
| **DATA** | Data integrity issue |

### Status Values

| Status | Meaning |
|---|---|
| **Open** | Issue identified, not yet addressed |
| **In Progress** | Fix being developed |
| **Fixed** | Fix implemented, awaiting verification |
| **Verified** | Fix confirmed working |
| **Won't Fix** | Decided not to address |
| **Duplicate** | Duplicate of another issue |
